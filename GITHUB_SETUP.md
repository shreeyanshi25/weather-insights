# GITHUB_SETUP.md — Make This Repo Recruiter-Ready on GitHub

A step-by-step guide to present your `weather-insights` project professionally on your GitHub profile.

---

## 1. Repo Description

When you edit the repo on GitHub, set the **Description** field to:

```
A React dashboard merging concurrent REST API streams from OpenWeather and NewsAPI to provide real-time meteorological conditions and aggregated local journalism for global cities.
```

Keep it under 350 characters. Lead with what it does, end with the tech stack keywords.

---

## 2. Add Topics / Tags

Go to your repository → click the **⚙️ gear icon** next to "About" on the right sidebar → add these topics:

```
react
vite
api-integration
openweather-api
rest-api
frontend
javascript
css
weather-app
news-api
```

> ✅ Topics make your repo easily discoverable and establish the breadth of technologies you employed.

---

## 3. Pin the Repo on Your Profile

1. Go to your **GitHub profile page** (github.com/shreeyanshi25)
2. Click **"Customize your pins"** (top of the pinned section)
3. Check ✅ `weather-insights`
4. Click **Save pins**

---

## 4. Add Live Demo URL

Since you deployed this to Vercel, make sure the Vercel link is set as the "Website" link!
In the **About** section (right sidebar of the repo):
- Click the **⚙️ gear icon**
- Paste your Vercel URL in the **Website** field.
- This creates a massive clickable link letting recruiters use the app instantly.

---
